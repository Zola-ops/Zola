## 功能概述

本工具包提供**端到端的自动化流程**，从原始聊天记录到最终 AI 人设 Skill：

```
原始聊天记录 → 解密数据库 → 导出CSV → 语言特征分析 → 生成人设MD
```

**三大核心功能**：

1. **数据提取**：自动扫描并解密本地微信/QQ 聊天记录
2. **语言分析**：统计分析个人的语言风格特征（口头禅、句式、标点等）
3. **人设蒸馏**：基于分析结果自动生成 AI 人设 Skill 模板

## 数据格式说明

### 1. 原始数据格式

#### 微信原始数据

**文件位置**：
```
旧版: C:\Users\<用户名>\Documents\WeChat Files\<wxid>\Msg\MSG0.db ~ MSG5.db
新版: C:\Users\<用户名>\xwechat_files\<wxid>_a47a\db_storage\message_0.db ~ message_3.db
```

**加密方式**：
- 算法：AES-256-CBC（旧版）/ SQLCipher（新版）
- 密钥：存储在微信进程内存中，需用 `pywxdump` 提取
- 消息格式：WCDB（WeChat Database） + Protobuf 压缩

**数据库结构**（解密后）：
```sql
-- 关键表：message_0 ~ message_3
CREATE TABLE message_0 (
    msgId INTEGER PRIMARY KEY,      -- 消息ID
    msgSvrId INTEGER,               -- 服务器消息ID
    type INTEGER,                   -- 消息类型（1=文本，3=图片，34=语音...）
    subType INTEGER,                -- 子类型
    isSend INTEGER,                 -- 是否发送（1=是，0=否）
    createTime INTEGER,             -- 创建时间（Unix时间戳，毫秒）
    talkerId INTEGER,               -- 会话ID
    content BLOB,                   -- 消息内容（Protobuf 编码）
    ...
);
```

#### QQ 原始数据

**文件位置**：
```
C:\Users\<用户名>\Documents\Tencent Files\<QQ号>\Msg3.0.db
C:\Users\<用户名>\Documents\Tencent Files\<QQ号>\Msg3.0.db.1.bak  （备份文件）
```

**加密方式**：
- 算法：SQLCipher（AES-256-CBC）
- 密钥：由 `KernelUtil.dll` 生成，通过 `key_func` 传入
- 密钥长度：16 字节
- 消息格式：QQ 自定义 MSG 序列化格式（以 `MSG\x00` 开头）

**数据库结构**（解密后）：
```sql
-- 两种表格式共存：
-- 格式1：buddy_<QQ号> / group_<群号>（下划线格式）
CREATE TABLE buddy_123456789 (
    Time INTEGER,                   -- 时间戳（秒）
    Rand INTEGER,                   -- 随机数
    SenderUin TEXT,                -- 发送者QQ号
    MsgContent BLOB,               -- 消息内容（MSG 格式）
    Info BLOB,                      -- 元数据（TD 格式）
    ...
);

-- 格式2：buddy$<QQ号>$Del$ / group$<群号>$Seq$（美元符号格式）
-- 仅索引，无消息内容
CREATE TABLE buddy$123456789$Del$ (
    Time INTEGER,
    Rand INTEGER,
    Seq INTEGER,
    ...
);
```

**MSG 格式说明**：
- 消息内容以 `MSG\x00` 开头
- 支持多种消息类型：文本、图片、语音、文件、表情等
- 文本消息直接存储明文
- 富媒体消息包含文件路径、缩略图等

### 2. 导出 CSV 格式

#### 统一格式（阶段一输出）

**文件名**：`wechat.csv` / `qq.csv` / `all_messages.csv`

**列定义**：

| 列名 | 索引 | 类型 | 说明 | 示例 |
|------|------|------|------|------|
| **时间** | 1 | 字符串 | 消息时间（YYYY-MM-DD HH:MM:SS） | `2024-01-15 14:30:25` |
| **平台** | 2 | 字符串 | 来源平台 | `微信` / `QQ` |
| **发送者** | 3 | 字符串 | 发送者昵称或QQ号 | `张三` / `123456789` |
| **是否我的发言** | 4 | 字符串 | 是否为当前用户发送 | `是` / `否` |
| **消息类型** | 5 | 字符串 | 消息类型 | `text` / `image` / `voice` / ... |
| **内容** | 6 | 字符串 | 消息文本内容（富媒体为空） | `哈哈哈哈好啊` |

**示例数据**：
```csv
时间,平台,发送者,是否我的发言,消息类型,内容
2024-01-15 14:30:25,微信,张三,是,text,哈哈哈哈好啊
2024-01-15 14:30:30,微信,李四,否,text,你在干嘛
2024-01-15 14:31:00,QQ,123456789,是,text,笑死我了
2024-01-15 14:31:05,QQ,987654321,否,image,
```

**注意事项**：
1. 时间格式统一为 `YYYY-MM-DD HH:MM:SS`
2. "是否我的发言"取值：`是` / `否`（也可用 `True`/`False`、`1`/`0`）
3. 富媒体消息（图片、语音等）的"内容"列为空
4. 文件编码：UTF-8（BOM 可选）
5. 换行符：Windows（`\r\n`）或 Unix（`\n`）均可

## 使用指南

### 场景 1：完整流程（首次使用）

```bash
# 步骤1：扫描本地聊天数据
python scripts/scan_chat_data.py
# 输出：数据文件路径列表

# 步骤2：解密微信数据库（如果已用 pywxdump 解密可跳过）
pywxdump bias_addr      # 从内存提取密钥
pywxdump db_decrypt     # 解密数据库
# 输出：decrypted/*.db

# 步骤3：解密 QQ 数据库（如果需要）
python scripts/hook_qq_key.py
# 按提示操作，输出：解密后的 db 文件

# 步骤4：导出为 CSV
python scripts/export_wechat.py --input decrypted/ --output wechat.csv
python scripts/export_qq.py --input decrypted.db --output qq.csv

# 步骤5：合并 CSV
python scripts/merge_export.py --wechat wechat.csv --qq qq.csv --output all_messages.csv

# 步骤6：语言特征分析
python scripts/analyze_speech.py --input all_messages.csv --output speech_report.txt

# 步骤7：蒸馏成人设 MD
python scripts/distill_persona.py --report speech_report.txt --output persona_skill.md
```

如果已经有导出的 CSV 文件（`all_messages.csv`），直接从步骤 6 开始：

## 技术原理

### 加密机制详解

#### 微信加密

1. **密钥存储**：密钥存储在微信进程内存中
2. **密钥提取**：`pywxdump` 通过内存扫描定位密钥偏移量
3. **解密过程**：
   - 使用密钥初始化 SQLCipher
   - 执行 `PRAGMA key = "..."`
   - 读取解密后的数据

#### QQ 加密

1. **密钥生成**：`KernelUtil.dll` 在首次打开数据库时生成密钥
2. **密钥传递**：通过 `key_func` 函数传入 SQLCipher
3. **密钥截获**：使用 Frida Hook `key_func`，在密钥传入时记录下来
4. **解密过程**：
   - 使用截获的密钥初始化 SQLCipher
   - 执行 `PRAGMA key = X'...'`（16字节密钥用 X'...' 格式）
   - 读取解密后的数据

### 消息格式解析

#### 微信 WCDB Protobuf

- WCDB 是微信自研的数据库格式（基于 SQLite）
- 消息内容使用 Protobuf 编码
- 需要解析 Protobuf 格式才能提取文本内容

#### QQ MSG 格式

- 消息内容以 `MSG\x00` 开头
- 支持多种消息类型：文本、图片、语音、文件、表情等
- 文本消息直接存储明文
- 富媒体消息包含文件路径、缩略图等

## 适用范围

### ✅ 支持的场景

| 场景 | 是否支持 | 说明 |
|------|---------|------|
| 本机完整流程 | ✅ 支持 | 从原始数据到最终人设 MD，一键执行 |
| 换一台自己的电脑 | ✅ 支持 | 需在新电脑重新执行**阶段一（数据提取）** |
| 分享给朋友使用 | ✅ 支持 | 朋友使用自己的聊天数据，跑相同脚本 |
| 直接拷贝解密后数据给朋友 | ❌ 不支持 | 涉及隐私，且朋友无你的密钥无法使用 |
| 跨平台（Windows→Mac） | ⚠️ 部分支持 | 阶段二/三（分析+蒸馏）跨平台，阶段一需适配 |

### 前置条件

**每台目标机器必须各自完成**：

1. 安装聊天软件（微信/QQ）需在本机安装并登录
2. 重新解密（密钥是**本机+本账号**特有的）
3. 导出 CSV（使用解密后的本地数据库）

### 可以跨机器共享的

- ✅ Skill 脚本（`scripts/` 目录）
- ✅ 技术文档（`SKILL.md`、`QUICKSTART.md`、`README.md`）
- ✅ 已导出的 CSV 文件（如果愿意分享）
- ✅ 已生成的人设 MD（如果愿意分享）

### 不可跨机器共享的

- ❌ 原始加密数据库（密钥不匹配）
- ❌ 密钥文件（`all_keys.json`、Frida Hook 抓到的密钥）
- ❌ 解密后的数据库（包含隐私，且绑定本机账号）

## 常见问题

### Q1: 提取到 0 条文本？

**可能原因**：
1. CSV 列索引不正确
2. "是否我的发言"列的值不匹配
3. 文件编码问题

**解决方法**：
```bash
# 检查 CSV 文件前几行
head -5 all_messages.csv

# 确认"是否我的发言"列的值
# 支持的取值：是、True、1、yes、Y
```

### Q2: 人设 MD 效果不好？

**可能原因**：
1. 样本量不足（建议 > 1 万条）
2. 缺少人设知识（职业、教育、兴趣等）
3. 模拟指南规则描述不准确

**解决方法**：
1. 增加样本量（导出更多聊天记录）
2. 手动补充人设知识图谱
3. 调整模拟指南的规则描述

2. 使用 `.bak` 备份文件触发解密流程
3. `KernelUtil.dll` 版本不匹配

**解决方法**：
```bash
# 使用 .bak 备份文件（会自动触发解密流程）
python scripts/hook_qq_key.py --input Msg3.0.db.1.bak

# 检查 KernelUtil.dll 版本
# 确保 Frida Hook 脚本中的偏移量正确
```
